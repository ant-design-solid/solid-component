Motion provides single-node enter/leave orchestration, while `MotionGroup` extends the same
lifecycle model to keyed lists.

## When to use

- You want lifecycle-aware transitions without committing to a prebuilt animation system.
- You need full control over the rendered element and the generated class names.
- You want a primitive that can be reused by popup layers, drawers or inline stateful content.

## Motion phases

### Appear and enter

When a node becomes visible, `Motion` progresses through prepare, start, active and end stages.
That gives you enough hooks to attach class names, force reflow and wait for motion completion.

### Leave

Leave follows the same staged lifecycle, then optionally removes the node when the transition
finishes.

## MotionGroup

Use `MotionGroup` when the visible state belongs to a collection instead of a single node.
Each item gets its own motion lifecycle, and `MotionGroup.Item` forwards the shared group config
so you only describe the keyed list and the rendered child.

- Pass `by` for object arrays so identity remains stable across inserts and removals.
- Apply the wrapper layout on `MotionGroup`, then render each animated DOM node with
  `MotionGroup.Item`.
- Current grouped motion covers appear, enter and leave. Reorder transitions are still out of
  scope.

## Notes

- `name` can be a string or a more explicit object when you need complete control over the class
  names for each phase and step.
- `removeOnLeave` determines whether the element is removed after the leave sequence completes.
- `MotionGroup` supports list `appear` / `enter` / `leave` with an optional `by` prop for stable
  identity. Primitive lists can omit `by`; object lists are expected to provide it.
- In development, `MotionGroup` warns when a key resolves to `undefined` or when duplicate keys
  are detected.
- `MotionGroup.Item` is the recommended way to bind grouped motion to DOM nodes without manually
  wiring `ref`, class names or styles.
- Reorder / move transitions are not implemented yet; current list support is limited to item
  insertion and removal.
