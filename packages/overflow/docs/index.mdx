Responsive composition with `Root`, `Prefix`, `Items`, `Item`, `Rest`, and `Suffix`.

## When to use

- You need to render a horizontal list that should gracefully collapse when width becomes constrained.
- You want to preserve leading and trailing affordances while only collapsing the middle items.
- You need more control than a one-off `overflow: hidden` treatment can provide.

## Composition model

### `Overflow.Root`

Provides the container and measurement context for the rest of the composition. This is where
`maxCount` and collapse behavior are configured.

### `Overflow.Items`

Owns the list iteration and lets `Overflow` keep track of the rendered items by `by`.

### `Overflow.Rest`

Renders the collapsed remainder. Use it when you want a `+3`-style indicator or a custom overflow
summary.

## Notes

- The package is designed for composition, not for shipping a styled overflow widget.
- Width measurement is only useful when the item subtree has stable layout semantics.
- If your list items can resize after mount, make sure the surrounding layout actually reflows.

<Callout tone="tip" title="Good fit">
  This primitive works well for selected-item chips, breadcrumb trails and segmented controls where
  preserving the leading or trailing affordances matters more than keeping every item visible.
</Callout>
